What happens when you initialize a repository? Why do you need to do it?It creates a .git folder and is ready for you to add files to the repository (but the repository is created empty)How is the staging area different from the working directory and the repository? What value do you think it offers?It allows you to operate on multiple files at once before committing them to the repository. This allows you to make sure you have everything you want before you commit (and can check with git status before you do)
